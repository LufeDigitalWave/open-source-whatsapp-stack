A complete, self-hosted WhatsApp business automation stack for teams that need full data control and powerful automation — no per-message fees, no vendor lock-in, no black-box APIs. One `docker compose up` and you have a production-grade inbox, CRM, and visual workflow engine running behind automatic HTTPS.

---

## What's Included

| Service | Version | Role |
|---|---|---|
| **Evolution API** | latest | WhatsApp multi-device connector (Baileys). Sends and receives messages via REST API. |
| **Chatwoot** | latest | Multi-channel inbox and CRM. Manages conversations, assigns agents, tracks contacts. |
| **n8n** | latest | Visual workflow automation. Build bots, lead capture, CRM sync, and anything HTTP. |
| **PostgreSQL** | 15-alpine | Shared relational database for all services. |
| **Redis** | 7-alpine | Job queues (Chatwoot/Sidekiq) and session cache (Evolution API). |
| **Traefik** | v3.0 | Reverse proxy. Automatic TLS via Let's Encrypt. Zero config for new services. |

## Architecture

```
                         Internet
                            |
                     :80 / :443
                            |
              +-------------+-------------+
              |         Traefik v3        |
              |  (TLS termination + route)|
              +--+----------+----------+--+
                 |          |          |
                 v          v          v
            +--------+ +------+ +----------+
            |Chatwoot| |  n8n | | Evolution|
            |  :3000 | | :5678| | API :8080|
            +---+----+ +--+---+ +----+-----+
                |          |          |
        --------+----------+----------+--------
                    internal network
                |                |
          +-----+----+    +------+---+
          |PostgreSQL|    |  Redis   |
          |   :5432  |    |   :6379  |
          +----------+    +----------+
```

**Message flow:** WhatsApp → Evolution API (WebSocket) → webhook POST → n8n → logic → Evolution API sends reply.

Full architecture explanation in [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).

## Quick Start

```bash
# 1. Clone
git clone https://github.com/LufeDigitalWave/open-source-whatsapp-stack.git
cd open-source-whatsapp-stack

# 2. Configure
cp .env.example .env
nano .env   # Set DOMAIN, ACME_EMAIL, SMTP vars

# 3. Point DNS before running (A records for all subdomains → VPS IP)
#    chatwoot.DOMAIN, n8n.DOMAIN, evolution.DOMAIN, traefik.DOMAIN

# 4. Run
chmod +x setup.sh
./setup.sh
```

The setup script generates all secrets, runs database migrations, and prints your service URLs.

## Included Workflows

Import these from the n8n UI (**Workflows → Import from file**):

### `workflows/echo-bot.json`

Replies to every incoming WhatsApp text message by echoing it back. Useful as a connectivity test or as a starting point for more complex bots.

**Nodes:** Webhook → Extract Message (Code) → Should Process? (If) → Send Echo (HTTP Request) → Respond 200

### `workflows/lead-capture.json`

Captures name and phone from incoming WhatsApp messages, upserts a record into PostgreSQL, and sends a confirmation reply.

**Nodes:** Webhook → Extract Lead Data (Code) → Should Process? (If) → Upsert in PostgreSQL → Send Confirmation (HTTP Request) → Respond 200

Before activating, run the schema migration:

```bash
docker compose exec -T postgres psql -U whatsapp -d n8n < workflows/leads-schema.sql
```

## Configuration Reference

All configuration is done via `.env`. See `.env.example` for the full list with comments. Key variables:

| Variable | Description |
|---|---|
| `DOMAIN` | Base domain. Services use subdomains of this. |
| `ACME_EMAIL` | Email for Let's Encrypt certificate registration. |
| `POSTGRES_PASSWORD` | Database password. Generated by setup.sh if left as placeholder. |
| `CHATWOOT_SECRET_KEY_BASE` | Rails secret key (64 hex chars). Generated by setup.sh. |
| `N8N_ENCRYPTION_KEY` | Encryption key for n8n credentials (32 chars). Generated by setup.sh. |
| `AUTHENTICATION_API_KEY` | Global API key for Evolution API. Generated by setup.sh. |

## Security Notes

- Change all `CHANGE_ME_*` values before exposing this to the internet (setup.sh handles this automatically)
- The `.env` file is in `.gitignore` — never commit it
- PostgreSQL and Redis are on an isolated internal network, not reachable from outside
- Restrict your VPS firewall to ports 22, 80, and 443 only
- Chatwoot account signup is disabled by default (`ENABLE_ACCOUNT_SIGNUP=false`)
